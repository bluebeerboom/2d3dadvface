## 修复签到标签跳动导致误签
- 问题：人脸出现初期相似度不稳定，匹配标签在多人之间跳动，导致误签他人
- 修复：添加帧级一致性确认机制（滑动窗口共识）
  - `RecognitionThread` 新增 `stabilize_frames=10` 参数
  - 按 bbox 中心网格化追踪人脸，每张脸维护 deque(maxlen=10)
  - 需连续10帧中 ≥9帧匹配同一人（且非Unknown）才触发签到
  - `_sign_emitted` set 防止重复emit
  - 定期清理失配的 face_key
- 效果：约0.3s确认延迟，彻底消除跳变误签

## 人脸识别准确度大幅提升 (v2)
- 问题：旧版 extract_gallery_features_realtime.py 不做人脸对齐，正确匹配相似度仅 ~0.35
- 根因：只用 bbox 裁剪后 resize 给 AdaFace，不同角度/姿态下特征差异巨大
- 改进 (extract_gallery_features_v2.py + realtime_face_recognition_v2.py)：
  1. **5点关键点对齐**：cv2.estimateAffinePartial2D 相似变换 → ArcFace 112x112 标准模板
  2. **双通道融合**：InsightFace ArcFace (主, 内置对齐) + 对齐后 AdaFace (辅), 加权融合
  3. **质量过滤**：det_score < 0.7 自动拒绝
  4. **鲁棒聚合**：≥5张用质量加权均值，<5张用中位数
  5. **自适应阈值**：top1/top2 差距大时降低阈值，差距小时提高
  6. **类间混淆检测**：自动报告高相似度 (>0.35) 的人员对
- 效果：类内相似度从 0.35 提升到 0.49-0.78 (ArcFace通道)
- 发现数据问题：Person_003 和 仲昭腾 是相同的5张照片 (MD5一致)，建议清理
- 新文件：galleries/gallery_arcface.npz, gallery_adaface.npz, gallery_meta.json
- 运行方式：python -u src/scripts/extract_gallery_features_v2.py (提取) → python -u src/scripts/realtime_face_recognition_v2.py (实时)
